# Vaultwarden Configuration

Sources: https://github.com/dani-garcia/vaultwarden/wiki/Configuration-overview and https://github.com/dani-garcia/vaultwarden/wiki/Disable-registration-of-new-users and https://github.com/dani-garcia/vaultwarden/blob/main/.env.template

## Overview

Vaultwarden can be configured via three methods (in order of increasing precedence):

1. **Environment variables** — recommended method
2. **`ENV_FILE`** — an environment file read directly by Vaultwarden at startup
3. **`config.json`** — generated via the `/admin` page; overrides all other settings (not recommended for manual editing)

The canonical reference for all options is the [`.env.template`](https://github.com/dani-garcia/vaultwarden/blob/main/.env.template) file.

---

## Configuration Precedence

```
Hardcoded defaults (src/config.rs)
    ↓ overridden by
ENV_FILE (/.env or configured path)
    ↓ overridden by
Environment variables
    ↓ overridden by
config.json (generated by /admin page)
```

> **Important:** Once you save settings via the `/admin` page, a `config.json` is created that takes highest precedence. Future changes via environment variables will have no effect for those settings unless you remove or edit `config.json`.

---

## Method 1: Environment Variables (Recommended)

Pass variables via `docker run -e`, Docker Compose `environment:` block, or a systemd `EnvironmentFile=`.

```bash
docker run -d --name vaultwarden \
  -e DOMAIN=https://vault.example.com \
  -e SIGNUPS_ALLOWED=false \
  -v /vw-data/:/data/ \
  -p 80:80 \
  vaultwarden/server:latest
```

Or in `compose.yaml`:

```yaml
services:
  vaultwarden:
    image: vaultwarden/server:latest
    environment:
      DOMAIN: "https://vault.example.com"
      SIGNUPS_ALLOWED: "false"
```

> **Note:** If you change an environment variable, you must **recreate** the container (not just restart) for the change to take effect.

---

## Method 2: ENV_FILE

Mount an environment file to `/.env` inside the container. Vaultwarden reads this file directly at startup. Values set in the actual environment still take precedence.

```bash
docker run -d --name vaultwarden \
  -v /path/to/vaultwarden.env:/.env \
  -v /vw-data/:/data/ \
  vaultwarden/server:latest
```

This is different from Docker's `--env-file` flag (which sets container environment at creation time), because Vaultwarden reads the file at runtime. This is useful for dynamic secret injection.

---

## Method 3: Loading Secrets from Files (`_FILE` suffix)

Any configuration option can be loaded from a file by appending `_FILE` to the variable name and pointing it to a file path. This is useful for Docker secrets.

```bash
# Instead of SMTP_PASSWORD=mysecret, use:
SMTP_PASSWORD_FILE=/run/secrets/smtp_password
```

The value stored in the file is used as the option's value, without exposing it as an environment variable.

## Key Configuration Options

### Domain (required for most features)

```env
DOMAIN=https://vault.example.com
# With non-default port:
DOMAIN=https://vault.example.com:8443
# With sub-path:
DOMAIN=https://host.example.com/bitwarden/
```

Always set `DOMAIN` to the full base URL. Attachments, invitations, and other features break without it.

### Signup and Invitation Controls

```env
# Disable new user registration (recommended after initial setup)
SIGNUPS_ALLOWED=false

# Allow only specific email domains to register
SIGNUPS_DOMAINS_WHITELIST=example.com,example.org

# Disable email invitations
INVITATIONS_ALLOWED=false

# Allow admin to invite users even when signups are disabled
# (This is always available via the /admin page regardless)
```

### Data Storage

```env
# Change the data folder location (default: ./data)
DATA_FOLDER=/var/lib/vaultwarden
```

### WebSocket Notifications

```env
# Enable real-time sync for desktop/browser clients
WEBSOCKET_ENABLED=true
```

### Logging

```env
LOG_LEVEL=warn        # Options: trace, debug, info, warn, error
LOG_FILE=/data/vaultwarden.log
EXTENDED_LOGGING=true # Include timestamps and log level in output
```

### Attachment Size Limit

```env
# Maximum size per attachment in KiB (default: 10240 = 10 MiB)
ATTACHMENTS_SIZE_LIMIT=10240
```

### Trash Auto-Delete

```env
# Days before items in trash are automatically deleted (0 = never)
TRASH_AUTO_DELETE_DAYS=30
```

## Docker Compose: Using a `.env` File

```
project/
├── .env
└── compose.yaml
```

**.env:**

```env
DOMAIN=https://vault.example.com
SIGNUPS_ALLOWED=false
ADMIN_TOKEN=mysecrettoken
```

**compose.yaml:**

```yaml
services:
  vaultwarden:
    image: vaultwarden/server:latest
    environment:
      - DOMAIN=${DOMAIN}
      - SIGNUPS_ALLOWED=${SIGNUPS_ALLOWED}
      - ADMIN_TOKEN=${ADMIN_TOKEN}
    volumes:
      - ./vw-data:/data
    ports:
      - 80:80
```
